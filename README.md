# Dopshy — QR Building Access Tracker

A Flutter mobile app for tracking employee entry and exit from buildings using QR code scanning. Users authenticate, set up a profile with a role, generate their personal QR code, and scan building QR codes to log access events.

---

## Features

| Feature | Description |
|---------|-------------|
| **Sign In / Sign Up** | Email + password authentication via REST API |
| **Profile Setup** | Full name, employee ID, department, photo, and role |
| **My QR Code** | Personal QR code containing the user's identity |
| **QR Scanner** | Camera-based scanner that logs building entry or exit |
| **Scan History** | Chronological list of all entry/exit events |
| **Roles** | Three access levels: User, Manager, Admin |
| **Theme** | Light / dark / system theme, persisted locally |
| **Offline storage** | All profile and scan data stored on-device with Hive (AES-256) |

## Getting Started

### Prerequisites

- Flutter SDK managed via [FVM](https://fvm.app)
- Xcode (for iOS builds)
- CocoaPods

### Install & run

```bash
# Install dependencies
fvm flutter pub get

# Run on iOS simulator (dev flavor)
fvm flutter run --flavor dev -t lib/main_dev.dart

# Run on iOS simulator (staging flavor)
fvm flutter run --flavor staging -t lib/main_staging.dart

# Run on iOS simulator (production flavor)
fvm flutter run --flavor prod -t lib/main_prod.dart
```

### Code generation

Run this after modifying any Riverpod provider, Freezed model, or Hive adapter:

```bash
fvm dart run build_runner build --delete-conflicting-outputs
```

## QR Code Format

### Building QR (scanned by users)

Generate a QR code from the JSON below using any QR generator. Each scan automatically alternates between **entry** and **exit** for the same building.

**Main Office — Floor 1**
```json
{"type":"building","buildingId":"B-001","buildingName":"Main Office","location":"Floor 1"}
```

**Server Room — Floor 3**
```json
{"type":"building","buildingId":"B-002","buildingName":"Server Room","location":"Floor 3"}
```

**Parking Garage**
```json
{"type":"building","buildingId":"B-003","buildingName":"Parking Garage","location":"Level B1"}
```

### User QR (generated by the app)

The **My QR** tab displays each user's personal QR code. Its content looks like this:

```json
{
  "type": "user",
  "userId": "<uuid>",
  "employeeId": "E-1001",
  "name": "Alex Johnson",
  "department": "Engineering"
}
```

## Project Structure

```
lib/
├── features/
│   ├── authentication/          # Login, register, credential caching
│   │   ├── data/                # HTTP auth repository, Hive credential store
│   │   ├── domain/              # LoginCredentials, LoginResponse (Freezed)
│   │   └── presentation/        # LoginScreen, RegisterDialog, LoginController
│   │
│   ├── user_profile/            # User profile management
│   │   ├── data/                # UserProfileRepository (Hive)
│   │   ├── domain/              # UserProfile (Freezed), UserRole enum
│   │   └── presentation/        # ProfileScreen, EditProfileScreen, ProfileController
│   │
│   ├── qr_scanner/              # QR scanning and history
│   │   ├── data/                # ScanHistoryRepository (Hive)
│   │   ├── domain/              # ScanRecord (Freezed), ScanType enum
│   │   └── presentation/        # ScannerScreen, MyQrScreen, ScanHistoryScreen
│   │
│   └── home/
│       └── presentation/        # MainScreen (bottom navigation shell)
│
├── router/                      # GoRouter configuration (SGRoute enum)
├── config/theme/                # ThemeLogic, FlexColorScheme setup
├── data/repository/             # NetworkRepository (Dio + interceptors)
├── hive/                        # Hive initialization and adapter registration
└── constants/                   # Endpoints, asset paths
```

## Tech Stack

| Layer | Technology |
|-------|-----------|
| State management | Riverpod 3 with code generation |
| Immutable models | Freezed + json_serializable |
| Navigation | GoRouter |
| Networking | Dio (retry + cache interceptors) |
| Local storage | Hive CE (AES-256 encryption) |
| QR scanning | mobile_scanner |
| QR generation | qr_flutter |
| Image picker | image_picker |
| Theming | FlexColorScheme (Material 3) |
| Localization | Easy Localization (EN, TR) |
